P2P File Sharing System - Sample Document
==========================================

This is a sample text document for testing the P2P file sharing system.

Features:
---------
1. Peer-to-Peer file transfer
2. SHA-256 hash verification
3. Piece-based downloading
4. Multiple concurrent peer support
5. Automatic retry of failed pieces

How it works:
-------------
The system breaks files into chunks (pieces) of 256KB each. Each piece is 
hashed using SHA-256 and stored in a .meta file. When downloading, each 
piece is verified against its hash before being saved to disk.

This ensures data integrity and allows resuming downloads from where they
left off (future enhancement).
